### Task 2: Type the render/copier boundary

Clears the entire 272-finding `Unknown` cascade in one logical change: basedpyright resolves pytest fixtures by *parameter name* and never infers a consumer param's type from a same-named fixture, so a nameable, importable `RenderFn` type must exist and be applied at every consumer. Two independent roots: the `render` fixture (39 consumers) and the one test that calls `copier.run_copy` directly (`template_root`).

**Why `Mapping[str, object]`, not `dict[str, object]`:** `dict` is **invariant** in its value type, so a `dict[str, object]` parameter rejects `MINIMAL` (inferred `dict[str, str | int | bool]`) with `reportArgumentType`. `collections.abc.Mapping` is **covariant** in its value type, so `Mapping[str, object]` accepts any `dict[str, V]` answer payload. It is also the *truthful* contract: `render` only reads its answers (it does `{**data}`), and `copier.run_copy`'s `data=` accepts a mapping. (This corrects the spec's Subdivision-1 recipe, which named `dict[str, object]` — verified to fail.)

### Task 4: Type the parse boundaries honestly (`reportAny` → 0)

The last 7 findings are values typed `Any` out of `yaml.safe_load`/`json.loads`/`re.findall`. A blanket `dict[str, object]` trips `reportIndexIssue` at the nested descents and `cast(..., Any)` trips `reportExplicitAny` (a `recommended` rule). The fix is **per-site truthful types**: `dict[str, object]` only where the test does membership/equality; precise `TypedDict`s where it descends into structure; `NotRequired` keys where the real workflow data is heterogeneous; and an **explicit `assert "env" in run_step`** that resolves the latent unchecked subscript at line 344 (turning a possible `KeyError` into a checked precondition) rather than dodging the diagnostic with a false required-key claim. Verified: this honest recipe reaches 0 with zero new rule names, and the runtime assertion always holds (the test already asserted `run_step["env"][...]`).

**Files:**
- Modify: `tests/test_generation.py`
